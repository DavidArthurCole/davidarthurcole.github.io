**Date:** 2026-04-29
**Status:** Approved, pending implementation plan

## Overview

Complete ground-up rebuild of davidarthurcole.me (formerly davidarthurcole.github.io) as a professional portfolio for David Cole, Applications Developer III and ServiceNow Architect at Rochester Institute of Technology. The site positions David 100% as a ServiceNow developer/architect with 6+ years of enterprise experience.

## Tech Stack

- **Astro 5** - static site generator, zero JS shipped by default
- **Vue 3** - interactive islands only, `client:load` or `client:visible` as appropriate
- **Tailwind CSS v3** via `@astrojs/tailwind`
- **TypeScript** strict mode throughout
- **Build output:** `docs/` (GitHub Pages compatibility during domain transition)

## sys_update_xml Compaction (pre-implementation)

The raw export at `sys_update_xml/sys_update_xml.json` is approximately 71 MB. Before building the site, `scripts/compact-sn-export.js` must be run once. It:

1. Stream-parses the file using a JSON streaming library (no full load into memory)
2. Filters to records where `sys_created_by` or `sys_updated_by` matches David's SN user
3. Strips all meta fields: `sys_mod_count`, `sys_tags`, `sys_domain*`, `checksum`, `payload` (raw XML blob), `sys_policy`, `sys_scope`, `sys_overrides`, and all other fields not in the keep list
4. Keep list per record: `name`, `sys_class_name`, `scope`, `description`, `category`
5. Writes `src/data/sn-export-compact.json` as a typed JSON array

The compacted file feeds `snProjects` in `site.ts`. Nothing else in the build reads the 71 MB original.

- Nav: hamburger collapse below `md`
- Hero: stacks vertically, PFP shrinks to 120px
- ServiceNow certs: horizontal scroll row on mobile, grid on `md+`
- Skills grid: 2-col mobile, 3-col `md+`
- Projects grid: 1-col mobile, 2-col `md+`
- Experience timeline: full width, left border retained

## Deployment

`astro.config.mjs`:
```js
export default defineConfig({
  site: 'https://davidarthurcole.me',
  outDir: './docs',
  integrations: [vue(), tailwind()],
});
```

GitHub Pages serves `docs/` on the `main` branch. When DNS points `davidarthurcole.me` to the Pages host, only the `site` config value needs updating - no structural changes.
